# Tokio Gemini Context

## Focus Area
**Async Runtime**

## Why Index?
Ensures the `op-workflows` and `op-gateway` handle async scheduling correctly. Fundamental for all async Rust operations in the ecosystem.

## Project Description
A runtime for writing reliable, asynchronous, and slim applications with the Rust programming language. It provides the building blocks for async I/O, networking, and scheduling.

## Key Concepts
- **Reactor/Proactor**: Event-driven architecture.
- **Tasks**: Green threads managed by the runtime.
- **Non-blocking I/O**: Efficient resource usage.

## Conventions
- **Language**: Rust
- **Style**: Asynchronous, Future-based.
- **Ecosystem**: Acts as the standard library for Async Rust.
